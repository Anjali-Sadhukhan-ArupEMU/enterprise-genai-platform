Option 2: Azure AI Foundry Project Access

Native RBAC but Wrong Control Surface

Pros

  • Native Azure RBAC
  • Native project access control

Limitations

  • Gives project visibility
  • Users may see deployments
  • Users may see assets and settings
  • Not least privilege
  • No budget enforcement
  • Limited governance
  • Solves project access, not consumption control

Stakeholder recommendation: Choose Option 1 only when speed is the priority and budget or identity-level tracking are not required. Choose Option 3 for enterprise governance and long-term control. In Option 3, token budgets are configured by admins for SDK and gateway runtime enforcement, not through chat onboarding.
