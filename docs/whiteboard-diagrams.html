Model Selection (Pending Customer Decision):
• Option A: User Choice — User picks model from dropdown (filtered by group). Simple, transparent. (4-8h effort)
• Option B: Auto-Routing — System analyzes query and picks best model from group's pool. Cost-optimized. (16-24h effort)

Chat History (Pending Customer Decision):
• Option A: Session-Based — Browser memory only. History lost on close. Cost: $0. (0h effort)
• Option B: Limited Persistent (Recommended) — Cosmos DB with TTL (30/60/90 days). Last 50 messages per chat. Cost: $5-25/mo. (8-12h effort)
• Option C: Full Persistent — Cosmos DB permanent. No limits. Cost: $25-100+/mo (grows). (12-16h effort)

Admin Flow: Admin assigns models + system prompts to Entra ID groups via config. System prompt applied automatically — users don't choose prompts.

Discussion Points for Customer:
• Redis Cache: Which queries are most repeated? Do we cache by exact match or semantic similarity?
• File Upload: What file types and max sizes? Should uploaded files persist or be session-only?
• Dictation: Browser-native Web Speech API (free, Chrome/Edge) vs Azure Speech Services (paid, all browsers)?
• APIM: Do we need per-user rate limits, or per-group? Should APIM also front the backend APIs or only AI model calls?
• Multi-Agent: What complex workflows justify agent orchestration? Do we need human-in-the-loop approval between steps?
• Web Search: Bing Search API or Bing Grounding in Azure AI? Should results be shown as citations alongside the answer?
